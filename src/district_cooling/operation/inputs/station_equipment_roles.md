# 小梅沙冷站设备角色说明

## 原设计结构

原设计为：

```text
1 台机载/基载冷水机组
3 台双工况冷水机组
1 套蓄冰槽
```

其中双工况冷水机组可以在制冷和制冰两种模式之间切换，制冰模式下出水温度更低，COP 通常低于常规机载冷水机组制冷。

## 当前实际结构

根据当前项目口径，实际运行结构为：

```text
2 台机载/常规制冷冷水机组
1 台双工况冷水机组
1 台故障双工况冷水机组
1 套蓄冰槽
```

原因是原 3 台双工况冷水机组中：

```text
1 台双工况冷水机组故障
1 台双工况冷水机组改造为机载/常规制冷
1 台双工况冷水机组仍保留双工况能力
```

冷却塔当前按设计和运行数据推断为：

```text
CT_01 ~ CT_09   双工况冷却塔，共 9 台
CT_10 ~ CT_12   基载冷却塔，共 3 台
```

这个判断来自当前实测数据的运行规律：

```text
双工况冷水机组制冰时，CT_01 ~ CT_09 的运行率约为 96% ~ 100%，CT_10 ~ CT_12 约为 65%。
机载冷水机组制冷时，CT_10 ~ CT_12 的运行率约为 96.7%，明显更像基载塔组。
```

其中 `CT_07` 在所有工况下运行率都很高，可能承担公共优先启用、轮值策略或测点保持等作用，建议后续结合现场控制逻辑进一步确认。

## 当前水泵测点角色推断

根据测点名称和不同工况下的运行率，当前水泵可以先按如下方式理解：

```text
PUMP_CHW_01        冷冻水泵/系统侧供冷循环泵
PUMP_CW_01         基载/机载冷水机组冷却水泵
PUMP_CW_DUAL_01    双工况冷水机组冷却水泵，制冰和双工况联合供冷时明显参与
PUMP_CW_DUAL_02    测点可疑：功率全程为 0，但频率全程为 59 Hz，暂不作为真实运行泵
PUMP_CW_DUAL_03    双工况冷水机组冷却水泵或公共双工况冷却水主泵，几乎全工况参与
PUMP_GLY_01        乙二醇/蓄冰侧制冰循环泵，制冰时最明显
PUMP_GLY_02        乙二醇/蓄冰侧释冰或板换取冷相关泵，加冰制冷时最明显
PUMP_GLY_03        乙二醇/蓄冰侧公共主泵，几乎全工况参与
PUMP_PHE_01        板换侧备用或测点异常，功率很低但频率有保持值
PUMP_PHE_02        板换侧释冰供冷泵，加冰制冷和联合供冷时明显参与
PUMP_PHE_03        板换侧释冰供冷泵，加冰制冷时明显参与，但频率测点偏可疑
```

这个判断来自以下规律：

```text
PUMP_CHW_01 和 PUMP_CW_01 在机载制冷、加冰制冷、联合供冷中运行率都很高，更像系统主循环和基载/机载冷却水泵。
PUMP_CW_DUAL_01 在双工况制冰时运行率约 94%，在联合供冷时约 100%，但在机载加冰制冷时几乎不开，更像双工况冷水机组相关冷却水泵。
PUMP_CW_DUAL_03 在各工况运行率都很高，可能是双工况冷却水主泵或共用泵。
PUMP_GLY_01 在双工况制冰时运行率约 95%，联合供冷时约 100%，更偏制冰/双工况侧。
PUMP_GLY_02 在机载加冰制冷时运行率约 67%，而制冰时几乎不开，更偏释冰供冷侧。
PUMP_PHE_02 和 PUMP_PHE_03 在机载加冰制冷时运行率较高，更偏板换释冰供冷侧。
```

水泵存在轮换和备用策略，以上结论应作为数据驱动推断；最终设备编号仍建议用现场泵房图纸或 PLC 点表确认。

## 冷机与水泵编号对应关系推断

进一步按启停同步率、功率相关性和工况同步性推断，当前可先采用如下编号对应：

```text
CH_01  ->  PUMP_CHW_01 + PUMP_CW_01
CH_02  ->  PUMP_CW_DUAL_01 + PUMP_GLY_01
CH_03  ->  PUMP_CW_DUAL_02（推断为故障或无效测点关联）
CH_04  ->  PUMP_CW_DUAL_03 + PUMP_GLY_03
```

对应依据：

```text
CH_01 与 PUMP_CW_01 的启停一致率约 99.0%，CH_01 状态与 PUMP_CW_01 功率相关系数约 0.968。
CH_01 与 PUMP_CHW_01 的启停一致率约 99.1%，但 PUMP_CHW_01 更像系统冷冻水总泵，不一定只服务 CH_01。

CH_02 与 PUMP_CW_DUAL_01 的启停一致率约 97.8%，CH_02 功率与 PUMP_CW_DUAL_01 功率相关系数约 0.951。
CH_02 与 PUMP_GLY_01 在 CH_02 运行时同步率约 100%，说明其与双工况/制冰侧高度相关。

CH_04 与 PUMP_CW_DUAL_03 的启停一致率约 99.7%，CH_04 状态与 PUMP_CW_DUAL_03 功率相关系数约 0.933。
CH_04 与 PUMP_GLY_03 的启停一致率约 99.6%，CH_04 功率与 PUMP_GLY_03 功率相关系数约 0.877。

PUMP_CW_DUAL_02 功率全程为 0，但频率全程为 59 Hz，结合当前存在 1 台故障双工况机组，暂推断其与故障机或无效测点相关。
```

板换侧泵不建议强行按冷机编号一一对应：

```text
PUMP_PHE_02、PUMP_PHE_03 更偏释冰/加冰供冷过程；
PUMP_PHE_01 更像备用或异常测点。
```

## 当前测点角色配置

工况识别脚本使用如下配置文件：

```text
src/district_cooling/operation/inputs/station_operation_mode_config.json
```

当前默认角色为：

```text
base_chiller_ids = CH_01, CH_04
dual_mode_chiller_ids = CH_02
ignored_chiller_ids = CH_03
```

含义为：

```text
CH_01  按机载/常规制冷冷水机组处理。该机可能对应由双工况改造而来的常规制冷机。
CH_02  按当前仍具备双工况能力的冷水机组处理，可用于制冰或制冷。
CH_03  暂不参与主工况判别。当前数据中功率测值长期很小，更接近故障机、无效主功率测点或非主出力状态。
CH_04  按机载/常规制冷冷水机组处理，是当前主要常规制冷出力设备。
```

如果后续确认具体编号与现场实际不同，只需要修改 `station_operation_mode_config.json`，不需要修改工况识别代码。
